4. The Most Important Strategic Decision

You should not make the entire website look like Amazon.


That would hurt your agency credibility.


Instead, use a hybrid model:


Homepage = strategic agency/product-led landing page

It should communicate trust, expertise, outcomes, and pathways.


/products = full commerce shop

This can behave like a digital marketplace.


/services = high-ticket conversion area

This should sell custom services.


/solutions = buyer-specific pages

These pages target different audiences and improve SEO.


Example:



“SaaS templates for startups”

“Business website templates for local businesses”

“AI automation services for small businesses”

“Technical SEO services for SaaS companies”


This structure lets you sell to both:



Technical buyers

Non-technical business owners

9. Database Design

Below is a practical Supabase schema.


profiles

Stores user profile data.


SQL
profiles
- id uuid primary key references auth.users(id)
- full_name text
- email text
- phone text
- company_name text
- website_url text
- role text
- country text
- created_at timestamp
- updated_at timestamp


products

Core product records.


SQL
products
- id uuid primary key
- sanity_id text
- slug text unique
- name text
- subtitle text
- description text
- category_id uuid
- product_type text
- status text -- draft, published, archived
- thumbnail_url text
- demo_url text
- repo_url text
- documentation_url text
- featured boolean
- created_at timestamp
- updated_at timestamp


product_variants

For Standard, Premium, Done-For-You.


SQL
product_variants
- id uuid primary key
- product_id uuid references products(id)
- name text -- Standard, Premium, Done-For-You
- price_cents integer
- currency text
- compare_at_price_cents integer
- license_type text
- includes jsonb
- delivery_type text -- instant_download, manual_delivery, service
- external_checkout_url text
- lemon_squeezy_variant_id text
- stripe_price_id text
- is_active boolean
- created_at timestamp


categories

SQL
categories
- id uuid primary key
- name text
- slug text unique
- parent_id uuid null
- description text
- sort_order integer


orders

SQL
orders
- id uuid primary key
- user_id uuid references profiles(id)
- email text
- status text -- pending, paid, failed, refunded
- payment_provider text -- stripe, lemon_squeezy, manual
- provider_order_id text
- subtotal_cents integer
- discount_cents integer
- total_cents integer
- currency text
- created_at timestamp


order_items

SQL
order_items
- id uuid primary key
- order_id uuid references orders(id)
- product_id uuid references products(id)
- variant_id uuid references product_variants(id)
- product_name text
- variant_name text
- price_cents integer
- quantity integer


downloads

Controls access to purchased files.


SQL
downloads
- id uuid primary key
- user_id uuid references profiles(id)
- order_item_id uuid references order_items(id)
- product_id uuid references products(id)
- file_path text
- download_limit integer
- download_count integer
- expires_at timestamp null
- created_at timestamp


leads

Every contact, WhatsApp click, bot inquiry, booking, and form should create or update a lead.


SQL
leads
- id uuid primary key
- email text
- name text
- phone text
- company_name text
- website_url text
- source text -- contact_form, ai_bot, whatsapp, booking, product_page
- intent text -- buy_product, customize_template, build_saas, seo_help
- budget_range text
- urgency text
- status text -- new, qualified, contacted, proposal_sent, won, lost
- score integer
- notes text
- assigned_to text
- created_at timestamp
- updated_at timestamp


lead_events

Track behavior.


SQL
lead_events
- id uuid primary key
- lead_id uuid references leads(id)
- event_type text -- viewed_product, clicked_whatsapp, submitted_form, booked_call
- page_url text
- product_id uuid null
- metadata jsonb
- created_at timestamp


service_requests

For detailed project inquiries.


SQL
service_requests
- id uuid primary key
- lead_id uuid references leads(id)
- service_type text
- project_description text
- selected_product_id uuid null
- selected_variant_id uuid null
- budget_min integer
- budget_max integer
- timeline text
- requirements jsonb
- status text -- new, reviewed, quoted, accepted, rejected
- created_at timestamp


appointments

SQL
appointments
- id uuid primary key
- lead_id uuid references leads(id)
- calendar_provider text
- provider_booking_id text
- scheduled_at timestamp
- timezone text
- status text -- scheduled, completed, cancelled, no_show
- notes text
- created_at timestamp


conversations

For AI bot/contact history.


SQL
conversations
- id uuid primary key
- lead_id uuid references leads(id)
- channel text -- ai_bot, whatsapp, email, contact_form
- status text
- summary text
- created_at timestamp
- updated_at timestamp


conversation_messages

SQL
conversation_messages
- id uuid primary key
- conversation_id uuid references conversations(id)
- sender_type text -- user, bot, admin
- message text
- metadata jsonb
- created_at timestamp


licenses

Useful for premium digital products.


SQL
licenses
- id uuid primary key
- user_id uuid references profiles(id)
- product_id uuid references products(id)
- order_id uuid references orders(id)
- license_key text unique
- license_type text -- personal, commercial, extended
- activation_limit integer
- activation_count integer
- status text -- active, revoked, expired
- created_at timestamp
