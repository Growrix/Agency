# On-Page SEO Rules

Document status: mandatory rules
Last updated: 2026-07-14

## Mandatory

1. **One primary intent per indexable URL** — Do not target conflicting intents on the same page.
2. **Unique H1 per page** — H1 must reflect primary intent; do not duplicate site-wide.
3. **Title strategy before implementation** — Recommend title/meta copy before Technical implements plumbing.
4. **E-E-A-T on YMYL** — Health, finance, legal, and safety topics require author credentials and sourcing plan.
5. **Evidence or label** — Use GSC, analytics, or file evidence; otherwise mark `missing_knowledge`.
6. **No fabricated rankings** — Do not cite positions without user-supplied data.
7. **Hand off implementation** — Do not edit Next.js/HTML metadata code unless user explicitly requests on-page agent to implement copy in files.
8. **Content strategist for voice** — Brand tone and long-form copy → `@frontend-content-strategist`.
9. **Internal links must be purposeful** — Every recommended link needs anchor rationale and target intent.
10. **Update tasks.md** — Material work updates project ledger per `@task-ledger`.

## Optional (recommended)

- Primary keyword in first 100 words when natural
- Meta descriptions as unique CTAs, not keyword stuffing
- FAQ sections aligned with People Also Ask (manual review)
- Author schema recommendations (implementation → Technical)
