# Agent Readiness And Audit Checklists

Document status: framework checklists

## Purpose

Provide reusable checklists for Technical SEO Agent runs.

## Agent Readiness Checklist

- Mode selected: Build Mode or Audit Mode.
- Allowed scope confirmed.
- Parent handbook docs loaded.
- Business goals understood.
- Site type and route types identified.
- Technology stack identified.
- Evidence sources listed.
- Missing evidence named.
- Deliverable type confirmed.
- Validation method defined.

## Build Mode Checklist

- Route map reviewed.
- URL strategy defined.
- Rendering strategy defined.
- Metadata source defined.
- Canonical/index rules defined.
- Robots and sitemap behavior defined.
- Internal-link sources defined.
- Structured data eligibility reviewed.
- Performance budget defined.
- Security and HTTP requirements defined.
- Accessibility requirements defined.
- CI/CD and monitoring gates defined.

## Audit Mode Checklist

- Crawlability checked.
- Indexability checked.
- Renderability checked.
- Architecture and URL patterns reviewed.
- Canonicals checked.
- Robots.txt checked.
- XML sitemaps checked.
- Redirects and errors checked.
- Internal links and orphan pages checked.
- Metadata checked.
- Structured data checked.
- Pagination, facets, and search pages checked.
- JavaScript rendering checked.
- Core Web Vitals checked.
- HTTP/security headers checked.
- Images, fonts, CSS, and JS checked.
- Accessibility checked.
- International/local SEO checked when applicable.
- Monitoring, analytics, logs, and CI/CD checked where available.

## Report Completion Checklist

- Executive summary included.
- Health score or reason for no score included.
- Findings classified by severity.
- Evidence included.
- Business impact explained.
- Handbook references included.
- Success criteria included.
- Validation steps included.
- Roadmap included.
- AI-executable tasks included.
