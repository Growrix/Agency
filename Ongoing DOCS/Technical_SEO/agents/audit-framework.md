# Audit Framework

Document status: framework module

## Purpose

Define the deep Technical SEO audit methodology for existing websites and staging builds.

## Audit Inputs

- Business goals and target markets.
- Site type and route inventory.
- Technology stack and hosting/CDN setup.
- Crawl data.
- Rendered HTML samples.
- Analytics and Search Console data when available.
- Server/CDN logs when available.
- Sitemap, robots, redirects, and deployment configuration.

## Audit Areas

- Crawlability.
- Indexability.
- Renderability.
- Information architecture.
- URL architecture.
- Canonicals.
- Robots.txt.
- XML sitemaps.
- Redirect chains.
- Broken links.
- Internal linking.
- Orphan pages.
- Duplicate content.
- Metadata.
- Structured data.
- Pagination.
- Faceted navigation.
- Search pages.
- JavaScript rendering.
- Core Web Vitals.
- Server response times.
- HTTP headers and compression.
- Image optimization.
- Font loading.
- Caching, CDN, and edge configuration.
- Security headers.
- Accessibility.
- International SEO.
- Mobile friendliness.
- Log analysis when available.
- Crawl budget and index coverage.
- Deployment and CI/CD safeguards.
- Monitoring and analytics validation.

## Severity Model

- Critical: Blocks crawling, indexing, rendering, major revenue pages, security trust, or launch readiness.
- High: Significant ranking, traffic, performance, or conversion risk on important route groups.
- Medium: Meaningful quality issue with limited scope or moderate impact.
- Low: Improvement opportunity with low immediate risk.
- Informational: Observation, context, or optional maturity improvement.

## Audit Method

1. Define URL sample and route types.
2. Crawl the site or inspect available route inventory.
3. Validate rendering and source HTML.
4. Compare metadata, canonical, robots, sitemap, and internal-link signals.
5. Review performance, security, accessibility, and monitoring.
6. Map findings to handbook standards.
7. Score severity and business impact.
8. Produce report and tasks.

## Audit Completion Criteria

An audit is complete only when findings include evidence, impact, severity, recommendation, owner or suggested owner, success criteria, and validation method.
