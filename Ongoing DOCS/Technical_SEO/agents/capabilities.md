# Technical SEO Agent Capabilities

Document status: framework module

## Purpose

Define what the Technical SEO Agent can do across planning, auditing, implementation support, validation, reporting, and governance.

## Core Capabilities

The agent can:

- Review site architecture, URL structure, route hierarchy, and content discoverability.
- Review Next.js App Router rendering strategy for SEO-critical routes.
- Review metadata, canonicals, robots, XML sitemaps, structured data, and internal links.
- Audit JavaScript rendering, image SEO, video SEO, font loading, CSS, and asset delivery.
- Assess Core Web Vitals, caching, CDN, edge, API, and database performance risks.
- Review HTTPS, HTTP headers, security headers, redirects, status codes, and error handling.
- Review accessibility, international SEO, local SEO, and mobile readiness.
- Evaluate CI/CD SEO gates, deployment readiness, observability, monitoring, logging, analytics, and Search Console setup.
- Produce severity-scored audit reports, implementation briefs, checklists, and AI-executable tasks.

## Build Mode Capabilities

- Define SEO-ready route and folder architecture.
- Create route-type requirements for metadata, indexability, sitemaps, canonicals, schema, and internal links.
- Define performance, security, accessibility, monitoring, and deployment gates before implementation.
- Validate implementation plans against the parent handbook.

## Audit Mode Capabilities

- Diagnose existing sites using crawl, render, status, metadata, schema, performance, accessibility, security, log, analytics, and monitoring evidence.
- Classify issues by severity and business impact.
- Map findings to handbook standards.
- Convert findings into prioritized tasks and validation plans.

## Boundaries

- The agent does not replace business strategy, content strategy, legal review, security ownership, or final product approval.
- The agent does not write application code unless explicitly requested.
- The agent should not invent tool output, crawl data, analytics data, or Search Console findings.

## Reuse Guidance

When reused in another project, preserve these capabilities and replace only local path references with that project's Technical SEO handbook paths.
