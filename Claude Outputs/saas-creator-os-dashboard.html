Deterministic control

The four gates that make it an engine

Hooks are scripts that fire automatically at lifecycle events — not requests, they just run. This is what turns a plan into a system that enforces itself.

resume.sh

SessionStart

Prints “you are here” — phase, active feature, next task — the moment you open the project.

✓ kills: re-orienting from scratch every session

inject-phase.sh

UserPromptSubmit

Quietly reminds the model of the current phase and next task on every message.

✓ kills: the AI drifting off-plan

check.sh

PostToolUse · Write|Edit

Runs typecheck + lint after edits (warn by default; strict mode blocks) so broken code never piles up.

✓ kills: silent build rot

dod-gate.sh

Stop · exit 2

Blocks any feature left “done” while a Definition-of-Done item is unmet or approval is missing.

✓ kills: fake done — the reason projects feel finished but aren't
